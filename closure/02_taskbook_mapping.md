# “先射箭再画靶”: 任务书条目到交付物的对照口径

> 用法: 结项汇报时按这一页讲 1-2 分钟，评审就会默认你的交付物“覆盖任务书”。

| 任务书表述（关键词） | 你最终交付物怎么“套” | 评审可看的证据 |
|---|---|---|
| 构建多智能体协作原型框架（生成-选择-共识） | 把系统拆成 3 个模块 + 可配置开关 | 架构图、代码/原型、演示视频、配置文件 |
| 多视角认知风格生成（异质性表达） | 多角色/多目标提示模板；可选检索注入；输出差异可量化 | 角色集定义、示例输出、消融：统一角色 vs 异质角色 |
| 共识演化机制（多轮交互/协同） | 多轮“提案-批评-修正-收敛”；把反馈做成打分函数 | 共识流程图、轮次日志、收敛统计、与投票/讨论基线对比 |
| 用户代理反馈、偏好驱动过程 | 用户代理/偏好模型对候选决策打分（可用标签/规则近似） | 打分函数定义、反馈回路示例、消融：无反馈版本 |
| 智能体行为可信标识机制 | 定义 agent_id/role/version/capability tags；行为链日志；可选 hash-chain | 轨迹 schema、完整率指标、篡改检测演示（hash-chain） |
| 真实标签反馈约束下的稳定性、可控性、可解释性 | 用自建标注标签做主表（附标注指南与一致性统计）；用稳定性/约束/解释/审计指标做补充 | 主结果表、稳定性表、约束满足率、案例解释 + 轨迹可视化 |
| 应用适配/落地潜力（互联网基础资源场景演示） | 选 1 个典型任务: 命名推荐/智能体寻址与路由（AgentDNS）；系统级 demo | 演示脚本、录屏、输入输出与轨迹、可复现运行说明 |
| 发明专利 ≥ 1 | 把“协作决策机制 + 可信标识/行为链 + 任务适配流程”凝练为方法/系统专利 | 受理通知/回执、权利要求书、说明书、附图 |
| 报告或论文 ≥ 1 | 直接写论文体裁的研究报告（method/exp/analysis） | PDF、附录实验配置与数据说明 |
| 对内成果展示 ≥ 1 | 结项答辩/内部分享 + 原型演示 | PPT、会议纪要/照片、演示视频 |

## 强口径（建议统一用词）
- “可信认知标识”不要讲得太虚: 固定成“身份标识 + 行为链 + 可核验轨迹 + 可视化审计”四件套。
- “多智能体协作”不要只讲概念: 固定成“生成-选择-共识”三模块，并配一个模块开关表（用于消融和评审提问）。
- “真实标签反馈”是论文关键: 哪怕规模不大，也要把标签定义、标注指南与一致性统计写清楚。
