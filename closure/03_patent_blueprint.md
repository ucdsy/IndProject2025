# 专利落地蓝图（从任务书到可提交文本）

> 目标: 3-4 周内形成“可提交的发明专利”文本与回执，同时让专利与论文互相不冲突。

## 1. 推荐选题（按稳妥程度排序）
### 选题 A（推荐，最贴任务书）
**一种面向智能体命名与寻址路由的多智能体反馈驱动共识决策方法、装置、设备及介质**
- 覆盖任务书关键词: 异质生成、用户代理反馈、共识演化、可信标识/行为链、真实标签约束验证。
- 好处: 既有“方法”也有“系统”，可用原型与实验支撑技术效果。

### 选题 B（备选，偏“可信标识”）
**一种智能体身份标识与行为链的可核验记录方法及系统**
- 覆盖: 可信标识/审计/溯源（更像“工程系统专利”）。
- 风险: 需要避免和通用审计/日志/区块链类已有方案撞得太明显，权利要求要写得更具体（与多智能体共识绑定）。

### 选题 C（你已有原型可走的方向）
**一种语义分层的智能体命名与解析系统（AgentDNS 类）**
- 覆盖: 标识（命名）、选择（路由/候选）、治理与审计。
- 风险: 更偏系统架构，需强调区别点（语义标签解析、可解释路由轨迹、公平调度、治理流程等）。

## 2. 选题 A 的技术方案骨架（可直接写进说明书）
### 2.1 要解决的技术问题
- 单一大模型在命名推荐/智能体寻址与路由决策任务中存在: 单视角、波动性、难以审计复核。

### 2.2 技术方案（方法流程）
1. 获取待决策样本 `x`（用户自然语言请求/上下文）与约束 `R`（命名空间与标签体系、可用智能体能力集合、公平/治理策略等）。
2. 基于预设的角色集合 `Roles={user,domain_expert,governance,risk,...}` 生成异质智能体集合 `Agents`，每个智能体具备不同的目标/偏好/提示模板。
3. 多轮交互生成候选路由/候选 fqdn 集合 `C_t`（每轮包含: 候选路径、理由、引用的标签/证据片段）。
4. 构建反馈函数 `F`:
   - 来自用户代理/偏好模型的偏好评分
   - 来自命名空间/标签一致性的约束评分
   - 来自成本/轮次的惩罚项
5. 基于 `F` 对候选路由打分并执行选择/聚合，得到共识路由决策 `y`；必要时进入下一轮修正直至满足收敛条件。
6. 生成并保存“可信标识与行为链记录” `T`:
   - 身份标识: agent_id、role、版本、能力标签等
   - 行为链: 每轮输入输出、候选、打分、选择理由的结构化日志
   - 可选: 对日志条目做 hash 链/签名，实现篡改可检测
7. 输出 `y` + 结构化解释 + 可视化轨迹（用于审计复核）。

### 2.3 有益效果（写法要“可验证”）
- 提升标签一致性/约束满足率（用你实验主表支撑）。
- 降低输出方差、提高稳定性（稳定性指标支撑）。
- 提供可复核的过程轨迹，便于审计（轨迹完整率、篡改检测 demo 支撑）。

## 3. 权利要求书（写作抓手）
> 下面是“结构化写法”，不是最终法律文本。交给代理人时你可以把变量替换成更具体实现。

### 独立权利要求 1（方法）
- 一种多智能体共识决策方法，其特征在于，包括:
  - 异质智能体生成步骤
  - 多轮交互生成候选路由步骤
  - 基于反馈函数对候选路由打分并形成共识的步骤
  - 生成并保存包含身份标识与行为链的可核验轨迹步骤

### 独立权利要求 2（系统/装置）
- 包括: 生成模块、交互模块、共识模块、可信标识模块、可视化模块、存储模块等，并说明模块间数据流。

### 从属权利要求（建议方向）
- 角色集合的具体构造方式（提示模板、检索注入）。
- 反馈函数的组成（偏好评分、标签一致性评分、成本惩罚）。
- 收敛条件（轮次上限、分数阈值、稳定性阈值）。
- 行为链结构（字段集合、hash 链计算方式、校验方式）。
- 在智能体命名推荐、智能体寻址与路由任务中的具体应用。
