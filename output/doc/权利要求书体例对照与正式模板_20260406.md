# 权利要求书体例对照与正式模板（2026-04-06）

## 1. 结论

当前 [权利要求书初稿_20260406.md](/Users/xizhuxizhu/Desktop/IndProj04/output/doc/权利要求书初稿_20260406.md) 的**技术内容方向基本对**，但**文稿体例仍偏内部草稿**，还不是最像正式提交件的状态。

最主要的问题不是“保护点错了”，而是“形式感还不够像专利局看到的成稿”。

## 2. 正式申请件的典型样子

### 2.1 形式上
- 不写 Markdown 标题；
- 不写“说明：本稿为内部版本”之类批注；
- 不写提纲性说明语；
- 每条权利要求独立编号；
- 每条权利要求通常是一个完整长句；
- 方法权利要求通常写成：
  - `1.一种……方法，其特征在于，包括以下步骤：……；……；……。`
- 从属权利要求通常写成：
  - `2.根据权利要求1所述的……方法，其特征在于，……。`
- 系统/装置权利要求通常写成：
  - `13.一种……系统，其特征在于，包括：……；……；……。`

### 2.2 用语上
- 尽量使用中文法律/专利表达；
- 术语要单一稳定；
- 主题名称必须清楚，不能写成模糊题目；
- 不写“为什么这样设计”“这样更好”之类说明性语言；
- 不写过强的工程内部代号，除非先定义并且后文统一使用。

## 3. 官方依据

### 3.1 CNIPA 对权利要求书的总要求
- 国家知识产权局公开问答写得很直接：
  - 权利要求书应当清楚、简要；
  - 权利要求书应当以说明书为依据；
  - 独立权利要求应当完整；
  - 权利要求应当满足单一性要求。

### 3.2 独立权利要求的常见体例
- 国家知识产权局公开培训材料给出的典型规则是：
  - 独立权利要求应当包括前序部分和特征部分；
  - 特征部分使用“其特征在于”或者类似表达；
  - 应当写明解决技术问题的必要技术特征。

### 3.3 从属权利要求的常见体例
- 国家知识产权局公开培训材料给出的典型规则是：
  - 从属权利要求应当包括引用部分和限定部分；
  - 引用部分写明被引用权利要求的编号及主题名称；
  - 限定部分写明附加技术特征。

### 3.4 方法/设备/介质的官方示例风格
- 国家知识产权局《专利审查指南》公开示例中，针对“去除图像噪声的方法”，示例体例就是：
  - `1.一种……方法，其特征在于，包括以下步骤：……`
  - `2.一种计算机装置/设备/系统，包括存储器、处理器……`
  - `3.一种计算机可读存储介质，其上存储有……`

## 4. 我们上一版为什么会让你觉得像提纲

### 4.1 确实像草稿的地方
- 文件顶部还有内部备注；
- 部分表达偏“说明会语言”，而不是“申请件语言”；
- `Stage R / Stage A / Stage B / Stage C` 这类项目内部代号用得偏多；
- 某些术语还没有完全做成“法律文本中的定义式表达”；
- 结构上虽然已经编号，但整体阅读感还是“研究方案拆解”，不像“专利权利要求成稿”。

### 4.2 其实已经接近正式件的地方
- 已经有独立权利要求和从属权利要求的层级；
- 方法、系统、设备、存储介质的保护主题是对的；
- 方法权利要求的主句结构已经接近正式样式；
- 从属权利要求也已经基本符合“根据权利要求X所述……”的惯例。

所以准确地说：
- **不是写错了方向**
- 而是**还没做完最后一层专利文风收束**

## 5. 我建议怎么改成更像正式申请件

### 5.1 题目和主题名称
- 保留：
  - `一种面向智能体语义寻址与执行发现的受约束多阶段路由决策方法`
- 不建议在权利要求里频繁混用：
  - `Stage A`
  - `Stage B`
  - `Stage C`
- 改为：
  - 快路径裁决步骤；
  - 慢路径共识步骤；
  - 实例筛选与排序步骤。

### 5.2 方法独立权利要求
- 保持“一条长句 + 若干分步骤”的正式体例；
- 每个步骤用分号收束；
- 最后一个步骤用句号结束；
- 删除不必要的解释性插话。

### 5.3 从属权利要求
- 尽量一条只附加一个层面的限定；
- 不要一条从属权利要求同时塞太多横向特征；
- 避免把说明书里的效果表达写进从属权利要求。

### 5.4 系统/设备/介质
- 系统权利要求可写模块；
- 设备和存储介质建议走最稳妥的标准写法；
- 如果后续希望，还可补一条“计算机程序产品”权利要求。

## 6. 可直接套用的正式模板

### 6.1 方法独立权利要求模板

`1.一种【发明名称】方法，其特征在于，包括以下步骤：`

`S1、接收……；`

`S2、基于……生成……；`

`S3、在……条件下执行……，得到……；`

`S4、基于……确定……；`

`S5、输出……。`

### 6.2 方法从属权利要求模板

`2.根据权利要求1所述的【发明名称】方法，其特征在于，所述……包括……。`

`3.根据权利要求1所述的【发明名称】方法，其特征在于，所述……至少基于……进行判断。`

`4.根据权利要求2或3所述的【发明名称】方法，其特征在于，……。`

### 6.3 系统独立权利要求模板

`13.一种【发明名称】系统，其特征在于，包括：`

`候选生成模块，用于……；`

`快路径裁决模块，用于……；`

`慢路径共识模块，用于……；`

`实例筛选与排序模块，用于……；`

`轨迹记录模块，用于……。`

### 6.4 设备/介质模板

`15.一种电子设备，其特征在于，包括处理器和存储器，所述存储器中存储有计算机程序，所述计算机程序在所述处理器上运行时，使所述电子设备执行权利要求1至12任一项所述的方法。`

`16.一种计算机可读存储介质，其上存储有计算机程序，其特征在于，所述计算机程序被处理器执行时，实现权利要求1至12任一项所述的方法。`
