## 3. holdout3 协作消融图

![Holdout3 collaboration ablation](/Users/xizhuxizhu/Desktop/IndProj04/output/figures/retrospective_train_test_20260331/03_holdout3_collaboration_ablation.png)

可直接引用的结论：
- 原始 gate 下，`single_v2 / homogeneous_v2 / heterogeneous_v2 / heterogeneous_v3` 在 `holdout3 test=80` 上最终都落在 `0.8625`。
- `heterogeneous_v3` 解决了旧 `heterogeneous_v2` 的退化问题，但原始 gate 仍未把职责化信息转成 test 提升。
- 接上 `aggressive replay` 后，`heterogeneous_v3` 在 `holdout3` test 子集达到 `0.9250`。

## 6. 执行负载代理图与表

![Execution proxy](/Users/xizhuxizhu/Desktop/IndProj04/output/figures/retrospective_train_test_20260331/06_execution_proxy.png)

> 说明：现有 trace 未稳定落 `latency/token_usage`，因此这里用 `slow_path_rate / stage_b_applied / changed / fixed / regressed` 作为执行负载与修正收益代理。

| 协议 | slow_path_rate | Stage B applied | Changed | Fixed | Regressed |
| --- | ---: | ---: | ---: | ---: | ---: |
| `single_v2` | 0.4525 | 181 | 3 | 3 | 0 |
| `homogeneous_v2` | 0.4525 | 181 | 2 | 2 | 0 |
| `heterogeneous_v2` | 0.4525 | 181 | 0 | 0 | 0 |
| `heterogeneous_v3` | 0.4525 | 181 | 2 | 2 | 0 |
| `heterogeneous_v3 + aggressive` | 0.4525 | 181 | **18** | **18** | **0** |

解读：
- 五种协议的慢路径覆盖率完全相同，差异不在“进没进 B”，而在“是否真的放行 override”。
- `heterogeneous_v3 + aggressive` 的收益来自 gate 释放，不是额外增加慢路径调用次数。
